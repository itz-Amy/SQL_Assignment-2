To maintain this as a materialised view the view would need to be regularly updated to match any of the data that was recently added to the base tables. 

Circumstances in insertion and deletion where the materialised view would be maintained without accessing the base table.

Insertion
a)The view would be maintained without accessing Part table if the record entered into the Part table has partCost > 1000 and then the partNo is added to the view. 
b) However, if partNo already exists in the view, then when a new record is added to the Part Table, no new row would be added to the materialised view. 
c) Lastly, if a record newly inserted to the Part table with partNo < 1000 it would not be added to ExpensiveParts.

Deletion
a) The view would be maintained without accessing Part table if the record deleted from the Part table is unique and only one of the partNo is in the table.
b) If the partNo deleted from the Part table has more than one PartNo in it(base table), when one is deleted, the row in the materialised view does not get deleted.
